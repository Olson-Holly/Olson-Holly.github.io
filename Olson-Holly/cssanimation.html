This example takes the circle and square and changes their original colors and then changes them to a different color. When it has run it will go back to its original color. I made it so that the circle and square will animate completly in 8 seconds.

This parallelogram changes as many colors as you would like to program it to by using percentages in the keyframe isntead of from and to. The parallelogram also moves intime to the percentages as you write the position in the code in the keyframe along with the color percentage.

Now you can make those moving shapes delay when they start and if you use a negative number to start the shape will not start at the beging, it will start in the frame at that negative point. All you have to do is use animation-delay. Here you see three objects, they all start their animation at different position because I gave them all animation delay. I also added animation-iteration-count to say how long I want them to do their animation. Because I have one animation-iteration-count as infinite it will continue to run as I do the program.
